# End-user Applications

Frontend and mobile applications for deEHR users.

> **Status:** Directory structure only (Phase 0). Implementation lands in
> **Phase 1+** as each component is built.

| App | Audience | Stack (planned) |
| --- | --- | --- |
| [`patient-web/`](patient-web/) | Patients (web) | TypeScript + Next.js, WCAG 2.1 AA |
| [`patient-mobile/`](patient-mobile/) | Patients (iOS/Android) | React Native |
| [`provider-portal/`](provider-portal/) | Hospitals, clinicians, insurers | TypeScript + Next.js |

Per-app frontend stack, accessibility conventions and design system
references will be documented inside each subdirectory's README once
implementation begins.
